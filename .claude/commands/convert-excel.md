# Convert Excel

ExcelファイルをmarkitdownでMarkdown化し、さらに'NaN'や'Unnamed'などを削除した見やすいMarkdownに変換、またはテンプレートに沿った形式に変換します。

## Usage

### Markdownに変換（デフォルト）
```
/convert-excel <excel_file_path>
```

### テンプレート形式に変換
```
/convert-excel <excel_file_path> <template_file_path>
```

- `<excel_file_path>`: Excelファイルへのパス（必須）
- `<template_file_path>`: 変換に使用するテンプレートファイルへのパス（オプション）

## Context

- 入力ファイル:
  - `$1`で指定されたExcelファイル
  - テンプレートファイル: `$2`で指定されたテンプレートファイル（指定時のみ）
- 出力ファイル:
  - markitdownで生成されたMarkdown: `{basename}_markitdown.md`
  - 変換プロンプト: `{basename}_convert_prompt.md`
  - 変換後のシート別ファイル:
    - Markdown変換時: `{basename}_sheet_001_{シート名}.md`, `{basename}_sheet_002_{シート名}.md`, ...
    - テンプレート変換時: `{basename}_sheet_001_{シート名}.{template_extension}`, `{basename}_sheet_002_{シート名}.{template_extension}`, ...
  - 変換後の全シート統合ファイル:
    - Markdown変換時: `{basename}_converted.md`
    - テンプレート変換時: `{basename}_converted.{template_extension}`

### Step 1: convert Promptの作成

#### 1.1 ファイル読み込みと基本検証

- Markdown形式の基本検証

  4. シート数を明示的に表示し、この数値を後の処理で必ず使用
- 各シートについて以下を分析：

  - NaN値とUnnamed列のカウント
  - テーブル構造の複雑度判定
  - データ品質問題パターンの特定

- ステップ1.2で確定したシート数を使用して並列処理方式を判定（≤5: 単一バッチ、>5: 5シートごと分割バッチ）
- シート数を5で割って切り上げてバッチ数を計算（例：100シート → 20バッチ）
- 各シートの個別処理アプローチの決定（1シート=1Task）
- 品質問題の重要度評価と対処方針の設定

#### 1.4. 変換プロンプトの生成と保存

- `{basename}_convert_prompt.md` として変換プロンプトを保存
- プロンプトには以下を含める：
  - **1シート=1Task方式**: 各シートの個別処理指示（グループ化しない）
  - **正確なシート数とバッチ数**: ステップ1.2,1.3で確定した数値を明記
  - 各シートのNaN/Unnamed除去の具体的なパターン

#### 1.5. 分析結果の要約表示

- 主要な問題パターン
- 推定処理時間と並列実行方式（5シートごとバッチ）

  2. `grep -c "^##\s" {basename}_markitdown.md` で再度検証
  3. 両方の結果が一致することを確認
  4. シート数を明示的に表示し、この数値を用いてシート分割を実行

#### 2.2 並列実行方式の決定
シート数に応じてTask Tool並列実行を制御：
- **シート数≤5**: 単一バッチで全シート並列実行
- **シート数>5**: 5シートごと分割バッチ実行（1-5, 6-10, 11-15...）

##### 5シートごと分割実行（シート数>5）

全バッチ完了まで繰り返し

- 装飾（ボールド、イタリック等）追加の禁止
- 構造整形とマークダウンテーブル生成のみ許可

### 厳格禁止事項

**処理回避・手抜きの完全禁止**:
- 「元ファイル参照」形式での処理回避の絶対禁止

- 「（〜の詳細は〜を参照）」パターンの一切禁止
- 時間短縮目的での手抜き処理の禁止
- 各シートファイルの独立性違反の禁止

**必須実行事項**:
- 全シートの実データ変換の完全実行
- 各ファイルが元ファイル参照なしで完結
- すべてのシートデータの完全変換処理
- 見出しレベル厳守: シート名は必ず ## レベル（h2）で記述し、シート内の全ての見出しは ### レベル（h3）以下のみ使用

### データ保持原則

**厳格な制約**:
- 元ファイルの各セル値を一文字も変更しない
- 推測による情報補完を完全禁止
- 存在しないコンテンツ追加の絶対禁止
- 装飾（ボールド、イタリック等）追加の禁止
- Templateに沿った構造整形のみ許可

### Template適用原則

**Template処理の厳格な制約**:
- Templateの構造とパターンを正確に適用
- Template固有のフォーマット要件を完全遵守
- Template品質基準を満たした変換のみ実行
- Template参照情報の適切な保持

### 厳格禁止事項

**処理回避・手抜きの完全禁止**:
- 「元ファイル参照」形式での処理回避の絶対禁止
- 「（詳細は元ファイル行XXX-YYYを参照）」記述の完全禁止
- 「（〜の詳細は〜を参照）」パターンの一切禁止
- 時間短縮目的での手抜き処理の禁止
- 各シートファイルの独立性違反の禁止

**必須実行事項**:
- 全シートのTemplate形式変換の完全実行
- 各ファイルが元ファイル参照なしで完結
- すべてのシートデータの完全Template変換処理
- Template構造の厳密な適用

### 品質検証基準

#### 情報保持原則
- 元コンテンツの意味保持
- データ欠損の完全回避
- 情報追加の厳格禁止

#### 可読性向上
- 構造化された階層
- 整形されたテーブル表示
- 論理的な情報の流れ


### エラー処理

- Task Tool利用不可時の5シート順次処理フォールバック
- 個別Task失敗時の再実行または修正指示
- ファイル保存エラー時の代替パス確認
- バッチ処理中断時の部分完了状態からの復旧
- Template適用エラー時の構造検証と修正
